# Employee - Project Overview

Employees focus on their own assignments, collaboration, reminders, reports, documents, and personal configuration.

![Employee Project Overview](/docs/screenshots/employee/project-overview.png)

## When To Use This Page

Use this page when you need to complete project overview work as the Employee role. The screenshot above was captured while signed in as this role, so the visible navigation and actions reflect the permissions available to that panel.

## Step By Step

1. Open Project Overview from the project page where available.
2. Review summary, financial, attachment, report, and timeline sections that are visible to your role.
3. Check client, PO, invoice, deadline, budget, and status data before making decisions.
4. Use tabs to move between overview information without changing the project board state.
5. If the page is blank or unavailable, return to the project board and choose a project explicitly visible to your role.
6. Use the overview as a read-and-verify screen before editing operational data elsewhere.

## Checks Before You Leave

1. Confirm the saved state appears on screen.
2. Refresh or revisit the page if the action affects counts, dashboards, or project boards.
3. Notify the next responsible person if the workflow now depends on them.
