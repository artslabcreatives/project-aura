# Admin - SSO Applications

Admins operate across the full Aura workspace: users, projects, tasks, clients, estimates, reports, documents, SSO applications, AI scenarios, and cross-department analytics.

![Admin SSO Applications](/docs/screenshots/admin/sso-applications.png)

## When To Use This Page

Use this page when you need to complete sso applications work as the Admin role. The screenshot above was captured while signed in as this role, so the visible navigation and actions reflect the permissions available to that panel.

## Step By Step

1. Open SSO Applications from the admin sidebar.
2. Create a new OAuth application only when the requesting system is approved.
3. Enter a recognizable name and exact redirect URI values.
4. Copy the client ID and secret immediately to the receiving system.
5. Regenerate secrets only during a planned rotation or suspected compromise.
6. Test the authorization flow after saving the client.

## Checks Before You Leave

1. Confirm the saved state appears on screen.
2. Refresh or revisit the page if the action affects counts, dashboards, or project boards.
3. Notify the next responsible person if the workflow now depends on them.
