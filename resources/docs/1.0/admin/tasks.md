# Admin - Tasks

Admins operate across the full Aura workspace: users, projects, tasks, clients, estimates, reports, documents, SSO applications, AI scenarios, and cross-department analytics.

![Admin Tasks](/docs/screenshots/admin/tasks.png)

## When To Use This Page

Use this page when you need to complete tasks work as the Admin role. The screenshot above was captured while signed in as this role, so the visible navigation and actions reflect the permissions available to that panel.

## Step By Step

1. Open Tasks from the sidebar.
2. Use filters/search to narrow the list by status, project, assignee, date, or priority.
3. Open a task to review its description, due date, assignees, attachments, comments, and history.
4. Update status or assignment only when the task is ready for that workflow change.
5. Use bulk update only after confirming every selected task should receive the same change.
6. Save changes and confirm the updated task appears in the expected column or list.

## Checks Before You Leave

1. Confirm the saved state appears on screen.
2. Refresh or revisit the page if the action affects counts, dashboards, or project boards.
3. Notify the next responsible person if the workflow now depends on them.
