# Admin - AI Scenarios

Admins operate across the full Aura workspace: users, projects, tasks, clients, estimates, reports, documents, SSO applications, AI scenarios, and cross-department analytics.

![Admin AI Scenarios](/docs/screenshots/admin/ai-scenarios.png)

## When To Use This Page

Use this page when you need to complete ai scenarios work as the Admin role. The screenshot above was captured while signed in as this role, so the visible navigation and actions reflect the permissions available to that panel.

## Step By Step

1. Open AI Scenarios from the admin sidebar.
2. Review existing sessions and scenario policies before starting a new one.
3. Start a session and describe the operational scenario clearly.
4. Attach supporting files only if they are safe and relevant.
5. Review generated actions or policies before activating them.
6. Activate only scenarios that have been checked against current business rules.

## Checks Before You Leave

1. Confirm the saved state appears on screen.
2. Refresh or revisit the page if the action affects counts, dashboards, or project boards.
3. Notify the next responsible person if the workflow now depends on them.
